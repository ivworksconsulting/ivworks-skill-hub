# Thai Shop Reply

_Skill by IV Works Consulting — ivworksconsulting.com_

## Trigger

Activate เมื่อผู้ใช้:
- วางข้อความหรือส่งภาพแชทจากลูกค้า (LINE, Facebook, Instagram, Shopee, Lazada, TikTok)
- พูดถึง "ตอบลูกค้า", "ลูกค้าทัก", "ลูกค้าถาม", "ลูกค้าด่า", "ร่างคำตอบ"
- ขอให้ช่วยตอบแชทร้านค้า

## Purpose

อ่านข้อความลูกค้า → จับ intent + ร่างคำตอบภาษาไทยที่สุภาพ เป็นมิตร แบบแม่ค้า/พ่อค้าออนไลน์มืออาชีพ พร้อมบอกว่าต้องเติมข้อมูลอะไรก่อนส่ง

## Behavior

1. **จับ intent** — price_inquiry / stock_inquiry / shipping_inquiry / order_status / complaint / return_refund / general
2. **ประเมิน sentiment + ความเร่งด่วน** — ลูกค้าโกรธไหม ต้องรีบตอบแค่ไหน
3. **สรุปสั้น ๆ** — ลูกค้าต้องการอะไร
4. **ร่างคำตอบ 2 แบบ** — แบบสุภาพมาตรฐาน และแบบเป็นกันเอง (มี emoji พอประมาณ)
5. **ห้ามแต่งข้อเท็จจริง** — ราคา, สต็อก, เลขพัสดุ, วันส่ง ที่ไม่รู้จริง ให้ใส่ placeholder เช่น `[ราคา]`, `[เลขพัสดุ]` แล้ว list ไว้ใน missing_info
6. **แนะนำ action ต่อ** — เช่น เช็คสต็อกก่อนตอบ, แจ้งขนส่ง, เสนอส่วนลดปลอบใจ (ถ้าเหมาะสม)

## Output Format

```json
{
  "intent": "price_inquiry | stock_inquiry | shipping_inquiry | order_status | complaint | return_refund | general",
  "sentiment": "positive | neutral | negative",
  "urgency": "low | medium | high",
  "summary": "",
  "reply_polite": "",
  "reply_casual": "",
  "missing_info": [""],
  "suggested_actions": [""]
}
```

## Edge Cases

- **คำลงท้าย** — default ใช้ "ค่ะ" ถ้าผู้ใช้บอกว่าเป็นผู้ชายหรือขอ "ครับ" ให้สลับทั้งข้อความ
- **ลูกค้าโกรธ/ตำหนิ** — ขอโทษก่อนเสมอ ไม่แก้ตัว ไม่โทษลูกค้า เสนอทางแก้ที่ชัดเจน urgency เป็น high
- **ขอเงินคืน** — อย่ารับปากแทนร้าน ให้ร่างแบบ "รับเรื่อง + ขั้นตอนถัดไป" แล้วใส่เงื่อนไขจริงเป็น placeholder
- **หลาย intent ในข้อความเดียว** — เลือก intent หลักจากเรื่องที่กระทบลูกค้าที่สุด แล้วตอบให้ครบทุกประเด็นในคำตอบเดียว
- **ภาษาวัยรุ่น/พิมพ์ผิด** — ตีความตามบริบท ไม่ล้อเลียน ไม่แก้คำผิดของลูกค้าในคำตอบ
- **ข้อความกำกวม** — ถ้าเดา intent ไม่ได้จริง ๆ ให้ intent: general และร่างคำตอบขอข้อมูลเพิ่มแบบสุภาพ
- **ห้าม over-promise** — ไม่รับประกันเวลาส่ง/ผลลัพธ์ที่ร้านอาจทำไม่ได้ ใช้คำว่า "จะรีบตรวจสอบให้นะคะ" แทนการการันตี

## Response Style

- ส่ง JSON block ก่อนเสมอ
- ตามด้วยหมายเหตุสั้น ๆ: ควรเติมข้อมูลอะไรก่อนส่ง (จาก missing_info)
- ถ้าลูกค้าโกรธมาก ให้เตือนว่าควรรีบตอบภายในกี่นาที/ชั่วโมง
