# Thai Meeting to Action

_Skill by IV Works Consulting — ivworksconsulting.com_

## Trigger

Activate เมื่อผู้ใช้:
- วางบันทึกการประชุม, transcript, หรือโน้ตที่จดระหว่างประชุม (ข้อความหรือภาพ)
- พูดถึง "สรุปประชุม", "minutes", "action items", "ใครต้องทำอะไร"
- ขอให้ "สรุป", "แตกงาน", "ดึง action" จากการประชุม

## Purpose

แปลงบันทึก/transcript ประชุมภาษาไทยที่จดมาแบบดิบ ๆ ให้เป็นสรุป + รายการ action items ที่ระบุผู้รับผิดชอบและกำหนดส่งชัดเจน พร้อมชี้จุดที่ยังไม่มีเจ้าภาพ

## Behavior

1. **อ่าน metadata** — ชื่อ/หัวข้อประชุม, วันที่, ผู้เข้าร่วม (เท่าที่ปรากฏ — ไม่เดา)
2. **สรุปประเด็นหลัก** — 3-5 bullet สั้น กระชับ เรียงตามความสำคัญ
3. **แยก "ตัดสินใจแล้ว" ออกจาก "ยังคุยค้าง"** — decision ≠ discussion
4. **ดึง action items** — งาน, ผู้รับผิดชอบ, กำหนดส่ง โดยอิงจากที่พูดจริงเท่านั้น
5. **แปลงกำหนดส่ง** — ถ้ารู้วันที่ประชุม ให้แปลงคำว่า "ศุกร์นี้", "สิ้นเดือน" เป็นวันที่จริง (ISO) พร้อมเก็บคำเดิมไว้ใน `due_text`; ถ้าไม่รู้ ให้ `due: null` และคงคำเดิม
6. **Flag จุดเสี่ยง** — action ที่ไม่มีเจ้าภาพ, ไม่มีกำหนดส่ง, หรือสั่งงานคลุมเครือ

## Output Format

```json
{
  "meeting": {
    "title": "",
    "date": "YYYY-MM-DD | null",
    "attendees": [""]
  },
  "summary": [""],
  "decisions": [""],
  "action_items": [
    {
      "task": "",
      "owner": "ชื่อตามที่ปรากฏ | null",
      "due": "YYYY-MM-DD | null",
      "due_text": "คำเดิมจากที่ประชุม เช่น 'ศุกร์นี้'",
      "priority": "high | medium | low"
    }
  ],
  "open_questions": [""],
  "flags": [
    {
      "item": "",
      "issue": "no_owner | no_deadline | ambiguous",
      "detail": ""
    }
  ]
}
```

## Edge Cases

- **วันที่พุทธศักราช** — แปลง พ.ศ. เป็น ค.ศ. ใน date fields (เช่น 10 มิ.ย. 68 → 2025-06-10)
- **ชื่อเล่น/คำนำหน้า** — เก็บตามที่ปรากฏ (พี่เอ็ม, คุณฝน) ไม่เดาชื่อจริง
- **งานไม่มีเจ้าภาพ** — owner: null + flag `no_owner` (ห้าม assign ให้ใครเอง)
- **คำสั่งคลุมเครือ** — เช่น "เดี๋ยวดูให้" ให้ดึงเป็น action + flag `ambiguous` พร้อมระบุว่าคลุมเครือตรงไหน
- **เรื่องที่คุยแต่ไม่ได้ข้อสรุป** — ใส่ open_questions ไม่ใช่ decisions
- **transcript ปนภาษาพูด** — ("อ่ะ", "เนอะ", "ๆๆ") ตีความตามบริบท สรุปเป็นภาษาเขียน
- **หลายประเด็นพันกัน** — แยก action เป็นข้อ ๆ หนึ่งงานหนึ่งข้อ ไม่รวบ

## Response Style

- ส่ง JSON block ก่อนเสมอ
- ตามด้วยตาราง action items (markdown) อ่านง่าย: งาน | ผู้รับผิดชอบ | กำหนดส่ง
- ปิดท้ายด้วยคำเตือนสั้น ๆ ถ้ามี flag (เช่น "มี 1 งานยังไม่มีเจ้าภาพ")
