# Thai Slip Extractor

_Skill by IV Works Consulting — ivworksconsulting.com_

## Trigger

Activate เมื่อผู้ใช้:
- ส่งภาพสลิปโอนเงิน / สลิปพร้อมเพย์ / สลิปจ่ายบิล จากแอปธนาคารไทย
- พูดถึง "สลิป", "สลิปโอนเงิน", "หลักฐานการโอน", "slip"
- ขอให้ "ตรวจสลิป", "อ่านสลิป", "ดึงข้อมูลจากสลิป" หรือ "เช็คยอดโอน"

## Purpose

แปลงสลิปโอนเงินจากแอปธนาคารไทย (ทุกธนาคาร + พร้อมเพย์) ให้เป็น JSON ที่นำไปใช้ต่อได้ทันที เช่น บันทึกยอดขาย, กระทบยอดบัญชี, ยืนยันการชำระเงินลูกค้า

## Behavior

1. **ระบุประเภทสลิป** — bank_transfer / promptpay / bill_payment / topup / unknown
2. **Extract ข้อมูลธุรกรรม** — วันที่+เวลา, จำนวนเงิน, ค่าธรรมเนียม, เลขที่อ้างอิง (transaction ref)
3. **Extract ผู้โอน** — ชื่อ, ธนาคาร, เลขบัญชี (ตามที่ปรากฏ — มักถูก mask บางส่วน)
4. **Extract ผู้รับ** — ชื่อ, ธนาคาร, เลขบัญชี/พร้อมเพย์ ID (ตามที่ปรากฏ)
5. **Flag ปัญหา** — ฟิลด์ที่อ่านไม่ออก, ข้อมูลขาด, สิ่งผิดปกติที่ควรตรวจซ้ำ
6. **เตือนข้อจำกัด** — การอ่านสลิปจากภาพ **ยืนยันไม่ได้ว่าเงินเข้าจริง** ต้องตรวจกับธนาคาร/statement เสมอ

## Edge Cases

- **วันที่พุทธศักราช** — แปลง พ.ศ. เป็น ค.ศ. (เช่น 15 มี.ค. 67 → 2024-03-15) รวมถึงชื่อเดือนย่อภาษาไทย (ม.ค., ก.พ., มี.ค., เม.ย., พ.ค., มิ.ย., ก.ค., ส.ค., ก.ย., ต.ค., พ.ย., ธ.ค.)
- **เลขบัญชีถูก mask** — เก็บตามที่ปรากฏ เช่น `xxx-x-x1234-x` ห้ามเดาเลขที่ถูกปิด
- **จำนวนเงินมี comma** — แปลง `1,250.00` → 1250.00 (number ไม่ใช่ string)
- **ไม่มีค่าธรรมเนียม** — ให้ fee: 0
- **ชื่อย่อธนาคาร** — ระบุชื่อเต็มถ้ารู้จัก (SCB → ไทยพาณิชย์, KBank → กสิกรไทย, BBL → กรุงเทพ, KTB → กรุงไทย, BAY → กรุงศรี, TTB → ทีทีบี, GSB → ออมสิน) ถ้าไม่แน่ใจให้ใส่ตามที่เห็น
- **สลิปปลอม/แก้ไข** — ตรวจไม่ได้จากภาพอย่างเดียว ห้ามรับรองความแท้ ใส่ verification_note เสมอ และ flag `suspicious` ถ้าเห็นสิ่งผิดปกติชัดเจน (ฟอนต์เพี้ยน, ตัวเลขซ้อน, layout ผิดจากปกติ)
- **ภาพไม่ชัด/ครอปขาด** — ลด confidence และ flag ฟิลด์ที่ไม่แน่ใจ

## Response Style

- ส่ง JSON block ก่อนเสมอ
- สรุปสั้น ๆ หลัง JSON: ใคร→ใคร, ยอดเท่าไหร่, เมื่อไหร่ + ปัญหาที่พบ (ถ้ามี)
- ถ้า confidence < 0.7 ให้แจ้งและขอภาพที่ชัดกว่า
- ย้ำเสมอว่าการอ่านสลิปไม่ใช่การยืนยันยอดเงินเข้า
